# Sprint 2 Planning

Plan Your Sprint

## Step 1: Select Sprint Work
As a team, discuss and select which user stories you'll work on in this Sprint. Consider:

Start with stories not completed last Sprint.
Move on to the next logical tasks in the project
Make sure not to overcommit on tasks/subtasks
Make sure everyone on the team has at least one task to work on
Consider dependencies—some features need to be built before others

- Step 2: Assign tasks to Team Members
- Move selected user stories from "Backlog" to "To Do" column
- For each user story, review the tasks and assign
- Make sure work is distributed fairly across the team
- Add "sprint-2" label to all Sprint issues and subtasks

## Submission Requirements

GitHub Project Board Link
- Screenshot of your configured project board with stories assigned to team members
- All user stories added as issues
- Sprint 2 items identified and assigned
