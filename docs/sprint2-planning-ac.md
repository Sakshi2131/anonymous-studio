# Sprint 2 Planning
--------------------

## Step 1: Select Sprint Work

As a team, discuss and select which user stories you'll work on in this Sprint. 

**Consider** :

1. Start with stories not completed last Sprint.
2. Move on to the next logical tasks in the project
3. Make sure not to overcommit on tasks/subtasks
4. Make sure everyone on the team has at least one task to work on
5. Consider dependencies—some features need to be built before others
6. Move selected user stories from "Backlog" to "To Do" column
7. For each user story, review the tasks and assign
8. Add "sprint-2" label to all Sprint issues and subtasks

## Submission Requirements

**GitHub Project Board Link** :

- Screenshot of your configured project board with stories assigned to team members
- All user stories added as issues
- Sprint 2 items identified and assigned
